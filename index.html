Database Concepts

Data

Data are the collection of raw facts and figures which are unorganized, uninterpreted, and isolated, which doesn’t give any particular sense. They are randomly gathered so that they can be processed to obtain the desired result.

Information

Information is the processed form of data. Information gives a meaningful result after data are interpreted. For example:

Ram, 17 year old and lives in Pokhara 18.

DBMS (Database Management System)

DBMS is a computerized record-keeping system. It is a software that defines, manipulates, and manages the database. It allows users to access files, update records, and retrieve data as requested.

Advantages of DBMS

  • Sharing Data
  • Reduce Data Redundancy
  • Data Backup and Recovery
  • Data Security
  • Multiple User Interface
  • Data Integrity

Disadvantages of DBMS

  • Expensive
  • Changing Technology
  • Backup is needed

Database Models

Hierarchical Database Model

This is one of the oldest database models, where data is represented in the form of records organized in a tree-like structure.

Network Database Model

The network model was designed to address the limitations of the hierarchical model by allowing many-to-many relationships.

Relational Database Model

In this model, data are organized into tables that contain rows and columns, also known as relations. It is the most common database model used today.

Structured Query Language (SQL)

SQL is a standard language for accessing and managing data in a database. It includes sub-languages such as DDL, DML, and DCL.

SQL Sub Languages: DDL, DML and DCL

1) DDL (Data Definition Language)

DDL is used by the database designer and programmers to specify the content and the structure of table. It is used to define the physical characteristics of record. It includes commands that manipulate the structure of object such as tables. For example, to create a table:

Syntax:

CREATE TABLE table_name (field_name1 data_type1, field_name2 data_type2, …);
CREATE TABLE Student (SN Number, Fname text);

2) DML (Data Manipulation Language)

DML is related with manipulation of records such as retrieval, sorting, display and deletion of records or data. It helps user to use query and display report of the table. It provides techniques for processing the database. It includes commands like insert, delete, select, and update to manipulate the information stored in the database.

Syntax:

INSERT INTO table_name VALUES (list of values);
INSERT INTO student VALUES (1, 'RAM');

3) DCL (Data Control Language)

DCL provides additional features for security of table and database. It includes commands for controlling data and access to the database. Some examples of these commands are GRANT, COMMIT, etc.

Entity Relationship Database Model (ER Model)

The entity relationship database model (ER diagram) is based on the perception of a real world that contains a collection of basic objects called entities and the relationships among these objects. The ER diagram is an overall logical structure of a database that can be expressed graphically. It was developed to facilitate database design. The major objective of the ER diagram is to show relationships among different entities. It has the following components:

1) Entity

The distinguishable object of this real world is known as an entity. It has a set of properties which uniquely identifies an entity. For example, if Student is an entity, then his/her name may be a property. It is denoted by a rectangle.

2) Attributes

Attributes are the properties possessed by an entity. They are represented by ellipses or oval shapes. For example, if Student is an entity, then its attributes can be registration number, name, roll no, class, address, etc.

3) Link

The flow of information is indicated by a link in an ER diagram. It is simply denoted by a line. It connects entities, attributes, and relationships.

4) Relationships

A relationship is an association among several entities. It is represented by a diamond shape. For example, if Teachers and Students are two entities, the association can be derived as: Teacher teaches Students. It shows meaningful dependencies between several entities. There are three types of relationships: One to One, One to Many, and Many to Many.

DBA (Database Administrator)

DBA is the most responsible person in an organization with sound knowledge of DBMS. He/she is the overall administrator of the program. He/she has the maximum amount of privileges for accessing the database and defining the roles of the employees who use the system. The main goal of the DBA is to keep the database server up to date, secure, and provide information to the user on demand.

Qualities of a Good DBA

  • Should have sound and complete knowledge about DBMS and its operations.
  • Should be familiar with several DBMS packages such as MS Access, MySQL, Oracle, etc.
  • Should have deep knowledge about the OS on which the database server is running.
  • Should have a good understanding of network architecture.
  • Should have good database designing skills.

Responsibilities

  • Install, monitor, and upgrade the database server.
  • Maintain database security by creating backups for recovery.
  • Conduct training on the use of the database.
  • Define user privileges, relationships, and manage forms and reports in the database.

Normalization

Normalization is the process of breaking down complex relations into simpler relations to reduce redundancy and ensure data integrity.

First Normal Form (1NF)

1NF eliminates duplicate columns and ensures each table has a primary key.

Second Normal Form (2NF)

2NF removes partial dependencies by placing data that applies to multiple rows in separate tables.

Third Normal Form (3NF)

3NF removes columns that are not dependent on the primary key and ensures that data is fully normalized.
